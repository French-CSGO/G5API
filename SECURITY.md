# Security Policy

## Supported Versions

The following table lists the versions of this project that currently receive
security updates. Only branches marked with :white_check_mark: are supported.

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please email us at
`security@example.com` with the details of your finding. If you prefer to use
a web form, you can also submit reports via our security contact form at
https://example.com/security.

When reporting a vulnerability, please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce, including any scripts, proof-of-concept code, or test data
- The affected versions, configuration, and environment details
- Any relevant logs or screenshots that help explain the issue

We will acknowledge receipt of your report within **3 business days**. You can
expect regular updates at least every **7 business days** while we investigate.
If we confirm the vulnerability, we will work to develop and test a fix, then
coordinate a release and disclosure timeline with you whenever possible.

We strongly encourage responsible disclosure. Please do not publicly disclose
the details of the vulnerability until we have had an opportunity to investigate
and release a fix or mitigations. We will credit you for the discovery if you
would like to be acknowledged in our release notes or advisories.
